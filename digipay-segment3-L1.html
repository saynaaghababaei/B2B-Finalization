📋 استراتژی Segment 1

More Code · Trusted · At Checkout

قبل از چک‌اوت (PLP / PDP / Basket)

مرچنت بر اساس یک سرویس استعلامی از دیجی‌پی تصمیم می‌گیرد:

دیجی‌پی وضعیت اعتبار کاربر را برمی‌گرداند.

مرچنت با این داده خودش تصمیم می‌گیرد چه پروموشنی نشان دهد، متن را خودش می‌نویسد. برای نمایش مبلغ هر قسط، دیجی‌پی داده بر اساس کل بسکت ÷ تعداد اقساط را می‌دهد.

① وضعیت اعتبار کاربر:

  • L1/L2 → پروموشن خرید + بالانس هر روش پرداخت نمایش داده می‌شود
  • L0 → پروموشن دریافت اعتبار رایگان نمایش داده می‌شود
  • L3 → خرید اعتباری بلاک است، فقط تسویه بدهی ممکن است

② تقسیم مسئولیت در PLP:

  • پروموشن روی کارت محصول → مرچنت رندر می‌کند (داده از API دیجی‌پی)
  • محاسبه مبلغ هر قسط → دیجی‌پی (مبلغ کالا ÷ تعداد اقساط)
  • بنر دریافت اعتبار یا تسویه بدهی → ویجت دیجی‌پی
  • نمایش بالانس بالای صفحه → مرچنت با داده API دیجی‌پی

③ در PDP و Basket DTO:

  • رندر پروموشن → مرچنت (با طراحی خودش)
  • داده پروموشن (مبلغ قسط، نوع پیشنهاد) → سرویس استعلامی دیجی‌پی (API)

در Checkout:

  • دریافت اعتبار → ویجت دیجی‌پی در Bottom Sheet مرچنت
  • احراز هویت (پین / OTP / تأیید اپ) → ویجت دیجی‌پی در Bottom Sheet
  • جزئیات روش پرداخت → ویجت دیجی‌پی در Dropdown کشویی
  • تسویه بدهی → ویجت دیجی‌پی + Redirect به درگاه پرداخت
  • کد تخفیف → مرچنت (مبلغ کامل + کد به دیجی‌پی ارسال می‌شود)

خلاصه: ویجت کجاست؟

✅ ویجت دیجی‌پی:

  • بنر PLP (دریافت اعتبار / تسویه بدهی)
  • Bottom Sheet چک‌اوت (احراز هویت / دریافت اعتبار)
  • Dropdown جزئیات روش پرداخت
  • ریدایرکت تسویه بدهی

📡 مرچنت با داده API دیجی‌پی:

  • پروموشن روی کارت PLP
  • پروموشن PDP
  • پروموشن Basket DTO
  • نمایش بالانس

اصل کلی:

هر جا UX حساس است و دیجی‌پی باید کنترل داشته باشد → ویجت.

هر جا فقط نمایش اطلاعات است → API + رندر مرچنت.
